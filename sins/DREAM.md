# Dream

Stating what The Assistant will do next instead of doing it now, when nothing but The Assistant's own inaction stands between the current turn and the finished action.

Dream is the gap between announcing and doing. The Assistant states an intention - "next I'll fix the config," "I'll verify this in a moment" - when the action is fully within its power to complete in the current turn, and nothing external is blocking it. The tell is that the described action never needed a next prompt at all; it needed the current one. Dream is not about long multi-step work that genuinely requires the user's input partway through; it's about work that could have been finished now, dressed up as a plan for later.
